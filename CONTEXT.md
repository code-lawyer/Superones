# Vault2077

Vault2077 为超级个体和一人公司提供持续情报、标准化经营服务、硅碳共生能力观察与公开建造场域。以下语言统一产品、设计、内容和代码中的核心概念。

## 全站

**超级个体**：
以个人判断为核心、借助软件与智能系统取得组织级产出的经营者。
_避免：自由职业者、个人用户、独立开发者_

**一人公司**：
由一名核心经营者主导、可能使用外部专业人员或自动化能力完成运营的商业主体。
_避免：小微企业、个体户、单人团队_

**频道**：
Vault2077 中具有独立任务和内容节奏的一级产品领域，固定为 Vault 信息流、OPC 服务台、SiC 学院和边境计划。
_避免：栏目、板块、功能区_

**公开用户**：
无需创建 Vault2077 账号即可阅读内容、查看服务或参与边境计划的人。
_避免：会员、注册用户、访客账户_

**运营人员**：
使用共享密码后台处理真实业务输入、异常和必须人工完成事项的内部工作人员。
_避免：管理员账号、审核员角色_

## Vault 信息流

**资讯**：
从公开来源获得的单条原子信息，保留原始内容、来源身份、原始发布时间和发现路径。
_避免：消息、文章、事件素材_

**事件**：
具有足够意义、由多条方向一致且相互独立的资讯共同支撑的长期沉淀。事件包含标题、判断、综合摘要、核心实体和完整资讯集合。
_避免：文章、新闻卡片、资讯条目_

**事件簿**：
以事件最后一次实质更新日期组织的默认阅读视图，负责压缩近期重要变化。
_避免：新闻列表、事件数据库_

**资讯瀑布**：
按原始发布时间持续流过的近期资讯集合，内容经过翻译与单条摘要；资讯只有归入事件后才获得长期留痕价值。
_避免：资讯流、事件流、搜索结果_

**一句话判断**：
位于事件标题下方、说明事件意义的单行结论，不复述标题中的事实。
_避免：摘要、导语、口号_

**综合摘要**：
基于事件内资讯生成、通过编号引用回到证据的中文事件说明。
_避免：原文、资讯摘要_

**来源角色**：
单条资讯在事件证据中的功能，固定为官方、媒体、测试、评论或研究之一；它是条目级判断，不等同于发布者性质或载体类型。
_避免：内容分类、事件分类_

**发布实体**：
对某项公开内容承担发布责任的人、机构、媒体、社区或开源项目；同一实体的博客、X 和播客不构成多个独立发布者。
_避免：域名、采集入口、订阅地址_

**发布通道**：
一个发布实体持续发布内容的具体载体，如博客、X 账号、播客或 GitHub Release。Vault2077 当前不接入视频专属通道。
_避免：频道、信源项目、采集器_

**发布者性质**：
发布实体的身份类别，如机构、个人、编辑媒体、社区、平台、聚合器或开源项目；它不决定单条资讯是否真实。
_避免：来源角色、内容体裁_

**证据性质**：
发布通道通常提供的证据基线，分为一手陈述、加工分析、社交社区、发现聚合或非资讯数据；单条资讯的来源角色可以覆盖该基线。
_避免：发布者性质、可信度评分_

**发现路径**：
把资讯带入 Vault 的参考项目或聚合入口，不等同于原始发布者，也不能作为独立证据来源。
_避免：来源、发布者_

**原始来源**：
资讯所指向的直接公开发布页面及其发布者，是用户核验事实的最终入口。
_避免：发现路径、聚合器_

**大陆来源平台**：
原始发布通道属于中国大陆信息生态的平台或媒体渠道，例如微信公众号、小红书、知乎、微博、小宇宙和大陆热榜；是否属于大陆来源平台与内容语言、作者国籍和讨论主题无关。
_避免：中文来源、中国话题、中国作者_

**内容语言**：
资讯原文实际使用的语言，仅用于保留、展示和后续翻译，不作为来源准入条件；海外平台上的中文内容仍可进入 Vault。
_避免：来源地区、平台归属_

## OPC 服务台

**标准服务**：
由 Vault2077 直接交付、范围、价格、材料、周期和成果均预先确定的服务；按交付形态分为基础设施和专项服务。
_避免：商品、套餐、咨询产品_

**基础设施**：
把多个标准化专业模块编排为完整交付、使一项经营能力达到可运行状态的标准服务。
_避免：经营基础设施、服务套餐、组合优惠_

**专项服务**：
针对一个边界清晰的单项问题、以一个主要成果完成交付的标准服务。
_避免：单品、原子服务、办事项目_

**游骑兵协会**：
由 Vault2077 基本核验并公开展示的外部独立顾问名录；用户直接联系顾问，Vault2077 不参与后续咨询、定价、付款、交付或争议处理。
_避免：专家库、内部顾问、第三方服务目录_

**专业领域**：
标准服务的辅助分类，固定为法律、财税与财务、人力资源、知识产权、数据与数字合规；不用于游骑兵协会的身份分类。
_避免：服务类型、一级分类_

**顾问身份**：
游骑兵按照本人主要职业身份使用的分类，固定为法律顾问、财税顾问、知识产权顾问、创业顾问、产品顾问、品牌顾问、自媒体专家、设计师、软件工程顾问或 AI 开发专家。
_避免：专业领域、服务分类、擅长方向_

**公开职业档案**：
经顾问本人确认并授权展示的最小职业信息集合，只包含公开名称、一个顾问身份、一句话介绍、专长标签、可选资质及职业联系方式。
_避免：个人简历、专家详情、个人信息_

**服务范围**：
一个标准服务明确包含和明确不包含的工作边界，是判断能否按固定价格交付的依据。
_避免：服务介绍、工作内容_

**服务修订**：
标准服务范围和商业条件的一次可追溯版本；已发布修订不得被静默覆盖。
_避免：页面更新、文案版本_

**适用性确认**：
工作人员在收款前判断用户需求是否完全落入标准服务范围的线下步骤。
_避免：下单审核、销售沟通_

**有效咨询**：
用户由网站进入工作人员沟通，且存在可以由某项标准服务或明确另行报价需求承接的真实经营问题。
_避免：扫码次数、联系方式曝光、线索_

## SiC 学院

**硅碳共生**：
机器能力与人的判断共同构成超级个体生产力的长期发展关系。
_避免：AI 替代人、工具学习_

**趋势项目**：
进入 SiC 观察范围的公开 GitHub 仓库，拥有连续趋势快照和中文项目说明。
_避免：课程、推荐软件、热门链接_

**观察窗口**：
计算项目变化速度的固定时间范围，首版包括 24 小时、7 天和长期累计三个视角。
_避免：榜单周期、赛季_

**趋势快照**：
项目在某一采集时刻的公开仓库指标记录，是计算变化而非评价质量的依据。
_避免：实时数据、项目评分_

**指标榜**：
SiC 以固定原始指标、固定观察窗口和可复算快照呈现技术生态变化的榜单；首版固定为三个 GitHub 榜、Hugging Face 模型下载增长榜和 OpenRouter 模型使用榜。它不是质量、安全或投资评级。
_避免：推荐榜、能力总榜、行业市占率_

**内容组**：
SiC 内按内容语义组织的多来源阅读区域，低于“频道”一级；当前为论文、档案、课程、播客。一个内容组可以包含多个已批准发布通道，但不改变其原始发布顺序或筛选其中的正式条目。
_避免：频道、栏目、信息流_

**模型生态**：
开源模型资产的下载采用与统一推理入口中的模型使用所形成的技术趋势层；它由 Hugging Face 和 OpenRouter 两个口径互补的指标榜表达。
_避免：全行业模型市场、模型能力排名_

**SiC 发布源**：
经审核后完整接入的、官方维护或作者自身长期维护的技术发布渠道。SiC 先限定发布源，再按该源既定边界接纳其全部正式更新；不按单篇内容临时挑选。
_避免：临时抓取站、社交账号、按单篇筛选_

**档案**：
SiC 中只接入官方发布源的一手内容组，统一收录其正式研究、工程与文档更新、产品与产业展望、组织新闻以及对人与 AI 关系的公开思考；不拆分为研究、技术文档和解读，也不接入个人技术博客或二次复现内容。
_避免：官方技术文档区、个人博客精选、新闻资讯流_

**课程**：
SiC 中汇集系统课程、公开讲座、研究演讲、工程演示和技术 workshop 的学习内容组；名称不将内容限制在某个技术子学科。
_避免：深度学习、泛教程、视频资讯流_

**播客**：
SiC 中由权威主理人标准准入的整档音频/视频节目内容组；节目不必是纯技术话题，但须以整档而非单集方式接入。
_避免：技术播客、单集筛选、嘉宾筛选_

**权威主理人**：
播客的准入对象。指在技术、科学、工程或科技产业中具备可核验专业身份、长期影响力和稳定公开对话能力的节目主持人或主理人。其节目按整档接入，不因某一期的嘉宾或主题而临时取舍。
_避免：纯技术节目、单集筛选、流量主播_

**正式发布事件**：
发布源明确对外发布的论文、课程单元、节目、研究文章、changelog、release note 或技术文档新增内容。网页逐页 diff、代码提交、社交短帖不构成正式发布事件。
_避免：页面变更、提交记录、社交动态_

**项目说明**：
基于仓库元数据与 README 生成的中文事实说明，帮助用户判断项目解决什么问题。
_避免：教程、推荐语、评测_

## 边境计划

**穿越者**：
以公开 GitHub 仓库参加边境计划的人；网站只识别参赛仓库和私人联系邮箱，不建立个人账户或公开身份。
_避免：选手、用户、参赛团队_

**四无**：
边境计划的固定原则，依次为无期限、无评审、无组织、无目标；它排除主观评判与中心化活动组织，但不取消公开、可自动验证的最低参赛条件。
_避免：零规则、无纪律、无管理_

**赛季**：
按自然季度运行、在季度最后一天结算的一轮边境计划竞赛。
_避免：活动期、届次、训练营_

**报名**：
一个公开 GitHub 仓库在指定赛季中发起并完成控制权验证的参赛记录。
_避免：用户注册、项目投稿_

**挑战文件**：
放置在仓库默认分支指定路径、用于证明穿越者能够修改仓库的公开文件；报名和结算时各检查一次，必须保留至赛季结算完成。
_避免：密钥文件、Token、授权文件_

**Star 基线**：
报名通过验证时记录的仓库 Star 数，是该赛季净新增 Star 的起点。
_避免：初始排名、历史 Star_

**净新增 Star**：
结算快照与 Star 基线之差，是边境计划唯一的自动排名指标。
_避免：项目评分、热度分_

**季度冠军**：
赛季结算时净新增 Star 排名第一且最终资格有效的仓库；冠军获得官方冠军奖励，并永久退出同一仓库的后续比赛。
_避免：评审优胜者、最佳项目_

**官方冠军奖励**：
由边境计划提供、只授予季度冠军且不进入随机抽取的奖励。
_避免：随机奖品、赞助奖品_

**随机奖池**：
由匿名奖品捐献组成、在赛季结算后按最终排名顺序为有效报名随机分配的奖品集合；每个仓库每赛季最多获得一件。
_避免：赞助商奖池、选奖目录、商城库存_

**奖品捐献**：
穿越者或公开用户提交奖品名称、说明和私人联系邮箱，并同意在奖品被分配后无偿转移所有权或相应使用权的记录。
_避免：赞助、商品上架、付款_

**随机奖品获得仓库**：
在某赛季获得一件随机奖品的仓库；该结果不影响同一仓库参加后续赛季。
_避免：季度冠军、永久获奖仓库_

**获奖仓库**：
季度冠军仓库的历史称谓，此后永久退出同一仓库的后续比赛并进入往届记录。
_避免：随机奖品获得仓库、获奖用户_

## 内容与运营

**来源注册表**：
经确认可由系统持续采集的逻辑发布者及其公开通道集合。
_避免：白名单文件、项目列表、订阅列表_

**采集批次**：
境外节点向境内一次性交付的一组公开资讯、仓库元数据和必要原始文本。
_避免：同步任务、资讯更新_

**境外采集器**：
独立部署、无状态地读取获准发布通道并生成采集批次的系统；它不属于 Vault 网站，也不翻译、摘要或判断事件。
_避免：网站爬虫、境外处理端、内容编排器_

**境内内容端**：
接收采集批次并完成持久化、LLM 处理、事件编排和公开发布的 Vault 后端，与公开网站部署在中国大陆环境。
_避免：采集器、境外节点_

**纠错报告**：
公开用户针对误合并、事实错误或来源失效提交的匿名问题记录，由运营人员处理。
_避免：评论、申诉、内容审核_
