# Vault2077 OPC 开发计划

> [!warning] 文档状态
> 本文固定 OPC 板块已经确认的内容开发方向，但不构成正式服务承诺。价格、地区、适用条件、交付周期、专业责任和正式服务范围仍须由对应专业负责人确认，并写入 [[Vault2077-OPC-Design-Spec]] 后方可进入生产状态。

## 1. 计划目标与边界

OPC 网站使用三个一级入口：

1. **基础设施**：把多个标准化专业模块编排为完整交付，使一项经营能力达到可运行状态。
2. **专项服务**：解决一个边界清晰的单项问题，以一个主要成果完成交付。
3. **游骑兵协会**：展示经过核验的外部独立顾问，为无法标准化的问题提供一对一咨询。

本文统一规划 OPC 的三个入口，不再为基础设施或游骑兵协会建立独立计划文件。当前已经确认十项基础设施、五个专项专业领域和十类游骑兵顾问身份。

网站负责解释服务、帮助用户判断适用性并完成联系交接；付款、材料接收、服务执行、客户管理和成果交付均在网站之外完成。

基础设施不是专项服务的折扣组合。每项基础设施必须拥有独立的目标、范围、实施阶段、整体价格、周期、成果、专业复核和服务修订。

专项服务不按文件类型、合同类型或办理动作无限拆分。每项专项服务必须对应一个用户愿意付费解决的专业问题，并以一个主要结果完成交付。

## 2. 基础设施准入标准

候选项目同时满足以下条件时，才进入基础设施目录：

- [ ] 解决一个完整经营结果，而不是单个手续。
- [ ] 至少需要两个相互依赖的专业模块。
- [ ] 大多数适用用户可以沿同一主流程完成。
- [ ] 开始条件、完成状态和验收成果可以明确说明。
- [ ] 创业者需要作出的关键决定可以提前列出。
- [ ] 超出标准路径的例外可以被识别并转交游骑兵。
- [ ] 价格、材料、周期、官方费用和第三方费用能够给出确定口径。
- [ ] 后续维护事项、有效期和复查触发条件能够说明。

## 3. 每项内容的统一开发模板

每个基础设施详情页必须完成以下内容：

- [ ] 服务名称、编号、专业领域和服务修订。
- [ ] 一句话说明最终建立的经营能力。
- [ ] 适用对象与不适用情况。
- [ ] 开始前状态与完成后状态。
- [ ] 包含模块与不包含事项。
- [ ] 用户必须作出的决定。
- [ ] 所需材料和材料齐备条件。
- [ ] 实施阶段、阶段成果和总体周期。
- [ ] 最终交付成果与验收方式。
- [ ] Vault2077 处理周期与官方或第三方周期。
- [ ] 整体价格、含税口径、官方费用和第三方费用。
- [ ] 关联专项服务。
- [ ] 转交游骑兵的例外条件。
- [ ] 交付后的维护事项、复查时间和失效条件。
- [ ] 对应专业负责人及复核类型。

## 4. 十项基础设施目录

| 编号 | 项目 | 建立的能力 | 开发优先级 |
| --- | --- | --- | --- |
| `OPC/INFRA/001` | 公司设立 | 使经营主体达到可以开始基本经营的状态 | P0 |
| `OPC/INFRA/002` | 财税运行 | 使收入、票据、支出、申报和关账形成周期性闭环 | P0 |
| `OPC/INFRA/003` | 商业交易 | 使报价、签约、交付、验收、开票和回款形成统一链条 | P0 |
| `OPC/INFRA/004` | 数字业务 | 使网站、App、AI 产品或在线服务具备基本公开上线条件 | P0 |
| `OPC/INFRA/005` | 核心资产 | 使品牌、代码、内容、域名和关键账号具备清晰权属与维护记录 | P1 |
| `OPC/INFRA/006` | 用工与协作 | 使员工、外包人员和长期协作者能够规范进入、工作和退出 | P1 |
| `OPC/INFRA/007` | 跨境 | 使合同、资金、税务和凭证能够共同解释一条具体跨境经营路线 | P0 |
| `OPC/INFRA/008` | 内容与直播 | 使创作者能够按确定收入模式开展直播、内容销售或商业合作 | P1 |
| `OPC/INFRA/009` | 行业上线 | 使特定行业和经营模式满足一套明确的上线条件 | P1 |
| `OPC/INFRA/010` | 暂停与退出 | 使经营活动、合同、人员、税务、数据和资产得到有序收束 | P2 |

## 5. `OPC/INFRA/001` 公司设立

**完成结果**：交付一个已经完成基础初始化、具备开始经营条件的经营主体，而不止是一张营业执照。

### 包含内容

- [ ] 主体类型、注册地和基本适用性判断。
- [ ] 名称、经营范围和主体登记。
- [ ] 章程及基础治理文件。
- [ ] 公章、财务章等印章事项。
- [ ] 银行开户材料准备或预约。
- [ ] 税务信息确认和发票能力初始化。
- [ ] 社保与公积金单位账户初始化。
- [ ] 证照、印章、银行和政务账号档案。
- [ ] 首年纳税申报、工商年报和重要期限日历。
- [ ] 开业后的第一次操作说明。

### 用户必须决定

- [ ] 主体类型、注册地、名称和经营范围。
- [ ] 股东、出资、法定代表人和基础治理安排。
- [ ] 是否需要立即开票、雇员或开展特殊许可业务。

### 游骑兵转交边界

- 复杂股权、代持、持股平台或融资架构。
- 外商投资、境外主体参与或 VIE 等跨境架构。
- 需要前置审批、特殊牌照或高度受监管的经营范围。

## 6. `OPC/INFRA/002` 财税运行

**完成结果**：建立按月或按季度持续运行的收入、票据、支出、申报和关账机制。

### 包含内容

- [ ] 财税基础信息和历史状态检查。
- [ ] 经营账户与个人账户的收支边界。
- [ ] 开票申请、开具、红冲和归档流程。
- [ ] 收入确认及平台结算记录口径。
- [ ] 报销、采购和费用凭证规则。
- [ ] 记账材料的固定收集方式和截止时间。
- [ ] 月度、季度和年度申报日历。
- [ ] 月度关账结果和待补材料清单。
- [ ] 应缴税费、现金余额和异常事项提示。
- [ ] 工商年报及年度税务事项。
- [ ] 创业者每期只需阅读的一页财税摘要。

### 用户必须决定

- [ ] 收入模式、定价方式和主要收付款渠道。
- [ ] 哪些费用属于经营支出。
- [ ] 是否接受发现的税务风险和整改方案。

### 游骑兵转交边界

- 历史账务重建、重大错报或长期未申报。
- 税务稽查、行政争议或需要正式鉴证意见。
- 复杂股权交易、跨境税务或特殊税收筹划。

## 7. `OPC/INFRA/003` 商业交易

**完成结果**：建立从报价、签约到交付、验收、开票和回款的统一交易链条。

### 包含内容

- [ ] 客户与供应商基础信息清单。
- [ ] 报价单、服务说明和工作范围模板。
- [ ] 保密协议。
- [ ] 常用服务、采购、合作或许可合同模板。
- [ ] 知识产权、保密、数据和责任条款。
- [ ] 签署主体、授权和电子签署规则。
- [ ] 预付款、里程碑、尾款和退款规则。
- [ ] 交付、变更和验收凭证。
- [ ] 开票信息与合同、回款之间的对应规则。
- [ ] 催款、逾期、暂停和终止模板。
- [ ] 合同、沟通和交易证据归档规则。

### 用户必须决定

- [ ] 产品或服务如何定价及承担何种商业风险。
- [ ] 哪些条款可以让步，哪些条款不能接受。
- [ ] 是否与某个具体客户或供应商建立关系。

### 游骑兵转交边界

- 融资、并购、重大知识产权许可或复杂跨境交易。
- 高额交易、行业特殊合同或责任无法标准化的项目。
- 已经发生违约、索赔、仲裁或诉讼的争议。

## 8. `OPC/INFRA/004` 数字业务

**完成结果**：使网站、App、AI 产品、SaaS 或在线服务具备一套可核对的基本公开上线条件。

### 包含内容

- [ ] 域名、经营主体和实际服务之间的关系检查。
- [ ] ICP 备案或相关许可路径判断。
- [ ] 用户协议及必要的交易规则。
- [ ] 隐私政策和个人信息收集清单。
- [ ] Cookie、SDK、模型服务和其他第三方处理方清单。
- [ ] 个人信息处理目的、方式和合法性基础。
- [ ] 同意、撤回及查阅、更正、删除等用户权利入口。
- [ ] 数据保存期限、删除和备份规则。
- [ ] 投诉、反馈和隐私联系渠道。
- [ ] 基础网络、账号和数据安全清单。
- [ ] 上线前检查和问题补正清单。
- [ ] 产品功能或数据处理发生变化后的更新规则。

### 用户必须决定

- [ ] 产品功能、目标用户和商业模式。
- [ ] 是否收集某类个人信息及是否使用第三方服务。
- [ ] 是否接受某项功能产生的数据与监管风险。

### 游骑兵转交边界

- 大规模处理敏感个人信息或发生数据安全事件。
- 重要数据、复杂跨境数据传输或强监管行业。
- 需要算法、深度合成、生成式人工智能等专项判断的产品。

## 9. `OPC/INFRA/005` 核心资产

**完成结果**：使品牌、代码、内容、域名和关键账号拥有明确的权属、证据、权限和维护安排。

### 包含内容

- [ ] 品牌、代码、内容、域名和账号资产盘点。
- [ ] 商标类别、申请顺序和续展日历。
- [ ] 软件著作权登记计划。
- [ ] 文字、图片、音视频、课程和数据内容的权属记录。
- [ ] 员工、外包人员和合作方成果转让文件。
- [ ] 开源软件许可证和使用义务清单。
- [ ] 字体、图片、音乐及其他第三方素材授权记录。
- [ ] 域名、代码仓库、应用商店和平台账号的主体归属。
- [ ] 管理员权限、双重验证和账号恢复方式。
- [ ] 创作过程、首次发布和版本证据归档。
- [ ] 核心资产总表及复查日历。

### 用户必须决定

- [ ] 需要保护的核心名称、成果和市场。
- [ ] 哪些资产归个人，哪些资产归经营主体。
- [ ] 是否允许合作方使用或再次许可某项资产。

### 游骑兵转交边界

- 已经发生权属争议、侵权、无效或异议程序。
- 国际商标、国际专利或多法域知识产权布局。
- 复杂开源合规、联合开发或重大技术许可。

## 10. `OPC/INFRA/006` 用工与协作

**完成结果**：建立员工、外包人员和长期协作者进入、工作、交付和退出的标准流程。

### 包含内容

- [ ] 员工、劳务、外包和供应商关系的基础判断。
- [ ] 招聘信息、录用通知和入职材料。
- [ ] 劳动合同、劳务协议或外包合同。
- [ ] 保密、知识产权和数据访问条款。
- [ ] 入职、账号开通、设备与资料交接。
- [ ] 薪酬、社保、公积金和休假接口。
- [ ] 基础工作规则和沟通机制。
- [ ] 外包工作范围、里程碑和成果验收。
- [ ] 离职、解除或终止合作文件。
- [ ] 账号、设备、资料和权限回收。
- [ ] 人员与协作者档案保存规则。

### 用户必须决定

- [ ] 是否雇佣或选择某个具体人员。
- [ ] 岗位、薪酬、工作方式和管理关系。
- [ ] 哪些数据、系统和商业信息可以授权访问。

### 游骑兵转交边界

- 高管、股权激励、合伙人或境外雇佣。
- 裁员、工伤、劳动争议或已发生的重大违纪。
- 用工关系无法明确、长期混同或存在历史风险。

## 11. `OPC/INFRA/007` 跨境

**完成结果**：针对一个明确交易场景，建立合同、资金、税务和凭证可以共同解释的跨境经营路线。

### 包含内容

- [ ] 交易场景、商品或服务类型及资金方向。
- [ ] 境内外交易主体和适用地区。
- [ ] 合同主体、币种和结算条款。
- [ ] 银行、支付机构、平台和账户路线。
- [ ] 交易真实性材料和留存要求。
- [ ] 发票、税务和收入确认口径。
- [ ] 结售汇、汇率和手续费记录。
- [ ] 退款、拒付和争议处理路径。
- [ ] 跨境数据和第三方服务初步判断。
- [ ] 定期对账、凭证归档和路线复查。

### 首批标准版本

- [ ] 境内主体向境外客户提供服务并收款。
- [ ] 海外平台收入回到境内。
- [ ] 境内主体采购境外软件或专业服务。
- [ ] 境外主体与境内经营者开展固定模式协作。

### 用户必须决定

- [ ] 进入哪个市场、采用哪个交易主体及承担何种商业风险。
- [ ] 使用何种收付款渠道并接受相应成本。
- [ ] 是否向境外提供数据、技术或持续服务。

### 游骑兵转交边界

- 多层主体、复杂受益所有权或无法解释的资金路线。
- 受制裁地区、特殊货物、虚拟资产或其他高风险业务。
- 跨境投融资、复杂常设机构或多法域税务安排。

## 12. `OPC/INFRA/008` 内容与直播

**完成结果**：使创作者能够按一种确定的收入模式开展直播、内容销售或商业合作。

### 首批标准版本

- [ ] 直播带货。
- [ ] 知识付费。
- [ ] 打赏与订阅。
- [ ] 品牌广告与商业合作。
- [ ] 课程、社群或咨询销售。

### 包含内容

- [ ] 收入模式、商品或服务类型确认。
- [ ] 经营主体、平台账号和实名信息。
- [ ] 收款、平台结算、开票和税务流程。
- [ ] 直播间、账号和内容基础规范。
- [ ] 广告、代言和营销内容标识。
- [ ] 商品或服务的准入与负面清单。
- [ ] 消费者权益、退款和售后规则。
- [ ] 图片、音乐、字体、片段和嘉宾授权。
- [ ] 品牌合作、经纪或联合制作合同。
- [ ] 评论、连线、敏感内容和突发事件处理。
- [ ] 收入、内容、授权和平台通知记录。

### 用户必须决定

- [ ] 创作主题、表达立场和内容方向。
- [ ] 接受哪些商业合作以及是否推荐某项商品或服务。
- [ ] 采用何种收入模式及承担何种公众责任。

### 游骑兵转交边界

- 新闻、医疗、金融、未成年人等特殊内容领域。
- MCN、经纪、多人主播或复杂平台合作关系。
- 已经发生封号、行政调查、重大舆情或消费者争议。

## 13. `OPC/INFRA/009` 行业上线

**完成结果**：针对一个确定行业、经营模式和地区，交付一套可以核对的上线条件和运行规则。

### 首批候选版本

- [ ] AI 应用上线。
- [ ] 开发者工具与 SaaS 上线。
- [ ] 在线教育或知识付费上线。
- [ ] 电商经营上线。
- [ ] 专业咨询业务上线。

### 每个版本包含内容

- [ ] 业务模式和适用范围定义。
- [ ] 主管部门、许可、备案和报告事项地图。
- [ ] 禁止、限制和条件准入事项。
- [ ] 必须公开的主体、价格、风险和联系信息。
- [ ] 用户协议、业务合同和内部规则。
- [ ] 广告、消费者权益、数据和知识产权检查。
- [ ] 日常操作、记录保存和异常处理流程。
- [ ] 检查、投诉或平台审核所需证据。
- [ ] 上线验收清单。
- [ ] 规则变化后的复查和修订机制。

### 用户必须决定

- [ ] 具体行业、产品功能、目标用户和收入模式。
- [ ] 是否取消或修改不符合准入条件的功能。
- [ ] 是否进入需要持续承担专业责任的业务。

### 游骑兵转交边界

- 无法归入已发布标准版本的行业。
- 医疗、金融等强监管业务。
- 需要申请特殊牌照、接受现场检查或形成正式法律意见。

## 14. `OPC/INFRA/010` 暂停与退出

**完成结果**：使停止、暂停或注销经营时的合同、资金、人员、税务、数据和资产得到有序收束。

### 包含内容

- [ ] 停止接单、停止续费和对外通知。
- [ ] 未完成合同、交付和退款安排。
- [ ] 应收、应付、库存和其他未结事项。
- [ ] 员工、外包人员和供应商关系终止。
- [ ] 税务检查、申报、清算和清税事项。
- [ ] 主体注销或暂停所需登记事项。
- [ ] 银行、支付、平台、邮箱和其他账号关闭。
- [ ] 用户数据、商业数据和个人信息的保存或删除。
- [ ] 域名、商标、代码、内容和设备等资产转移。
- [ ] 印章、证照和长期保存档案处理。
- [ ] 完成后的退出结果清单。

### 用户必须决定

- [ ] 暂停经营还是永久退出。
- [ ] 如何处理客户、员工、债权债务和核心资产。
- [ ] 哪些资料继续保存以及由谁承担后续责任。

### 游骑兵转交边界

- 资不抵债、无法清偿或进入破产程序。
- 存在诉讼、行政调查、重大税务风险或员工争议。
- 股东冲突、资产权属不清或跨境主体退出。

## 15. 专项服务准入标准

专项服务必须同时满足：

- [ ] 需要专业判断，而不是代填表或转交材料。
- [ ] 解决一个具体问题，并拥有一个主要交付结果。
- [ ] 材料、范围、价格和周期可以固定。
- [ ] 用户支付的是风险降低或决策质量，而不是普通渠道已经免费提供的操作。
- [ ] 不需要多个专业方向持续编排；需要编排时进入基础设施。
- [ ] 高度依赖个案判断、谈判或争议代理时可以转交游骑兵。
- [ ] 同类文件和事项已经合并为一个服务项目或有限的明确版本。

以下内容不作为独立付费专项服务：

- ICP 备案、App 备案和普通工商信息变更。
- 常规代理记账、机械申报和平台已有的标准操作。
- 每一种合同、作品、申请表或证明文件各建一个服务。
- 可以由一篇清晰免费指南解决的问题。

这些内容应成为免费办事指引、基础设施的组成模块，或专项服务中的具体交付文件。

## 17. 法律

法律方向覆盖经营风险识别、商业交易、公司治理、公开经营规则和履约异常，不等同于合同起草目录。

### `OPC/LEGAL/001` 经营法律体检

**主要结果**：一份经营法律风险分级报告和整改优先级。

- [ ] 检查主体与基础治理文件。
- [ ] 抽查常用合同和对外公开规则。
- [ ] 检查用工、外包和成果归属的明显缺口。
- [ ] 识别已知纠纷和潜在风险。
- [ ] 区分可由专项服务整改、需进入基础设施和需转游骑兵的事项。

本服务只负责固定范围诊断，不负责完成全部整改。

### `OPC/LEGAL/002` 单项商业合同专项

**主要结果**：一份可以使用的合同文本及关键风险说明。

- [ ] 在一个详情页内提供“审阅修订”和“标准起草”两个明确版本。
- [ ] 限定一份合同、常见商业交易和固定篇幅。
- [ ] 处理交易范围、价款、交付、验收、知识产权、数据、责任和终止条款。
- [ ] 包含一次固定范围修改。
- [ ] 提供履行合同需要注意的事项。

融资、并购、重大跨境交易、复杂谈判和争议合同转游骑兵，不按合同类型拆分服务页面。

### `OPC/LEGAL/003` 公司治理专项

**主要结果**：针对一次明确治理事项形成可签署或可归档的治理文件和权限说明。

- [ ] 检查章程与本次事项的关系。
- [ ] 制作股东决定、股东会决议或其他基础决策文件。
- [ ] 明确法定代表人、管理人员和对外签署权限。
- [ ] 处理一次简单治理信息调整所需文件。
- [ ] 说明文件签署、留存和后续办理要求。

复杂股权设计、控制权冲突、代持、融资和持股平台转游骑兵。

### `OPC/LEGAL/004` 商业规则与宣传合规审查

**主要结果**：一套经过修订、可以公开使用的商业规则或宣传文本。

- [ ] 检查服务条款、付费、退款和售后规则。
- [ ] 检查定价、促销、推荐和商业合作表述。
- [ ] 检查会员、订阅和自动续费说明。
- [ ] 检查广告、代言和营销内容标识。
- [ ] 交付问题清单和修订文案。

强监管商品或服务、重大公众争议和需要正式专项法律意见的事项转游骑兵。

### `OPC/LEGAL/005` 履约异常与争议前处理

**主要结果**：一份事实与证据清单、一份标准通知文件和下一步行动方案。

- [ ] 梳理一项合同关系的事实、时间线和现有证据。
- [ ] 识别尚未履行、履行不符或付款逾期问题。
- [ ] 制作催告、整改、暂停或终止通知中的一种。
- [ ] 说明可选沟通路径和主要风险。
- [ ] 给出继续协商或升级处理的条件。

已进入正式索赔、仲裁、诉讼或事实高度争议的事项转游骑兵。

## 18. 财税与财务

财税与财务方向不提供普通代理记账和机械申报，集中在发现问题、建立处理口径、清理混乱和支持经营判断。

### `OPC/FIN/001` 财税健康体检

**主要结果**：一份确定期间内的财税风险分级报告和整改清单。

- [ ] 检查申报、票据、收入和账务的一致性。
- [ ] 检查主体当前税务状态和明显异常。
- [ ] 列出缺失材料及补正优先级。
- [ ] 区分立即处理、持续观察和专业升级事项。
- [ ] 给出后续财税运行建议。

税务稽查、重大漏报和历史账务重建转游骑兵。

### `OPC/FIN/002` 主体与商业模式税负测算

**主要结果**：一份基于明确假设的主体、收入模式和税负比较。

- [ ] 限定一种主要商业模式和一组收入成本假设。
- [ ] 比较适用主体或结算方式。
- [ ] 测算主要税费与现金影响。
- [ ] 说明开票和收入确认口径。
- [ ] 给出推荐方案及其成立条件。

复杂股权、跨境税务架构和专项筹划转游骑兵。

### `OPC/FIN/003` 公私资金与股东往来清理

**主要结果**：一份资金往来分类表、凭证缺口清单和规范化方案。

- [ ] 识别个人账户和公司账户混用。
- [ ] 分类股东垫款、借款、报销、分红和其他往来。
- [ ] 整理平台收入进入个人账户等常见问题。
- [ ] 列出需要补充的凭证和说明。
- [ ] 给出调整步骤和后续收支规则。

无法解释的大额资金、涉嫌违法交易和重大税务风险转游骑兵。

### `OPC/FIN/004` 单项交易财税处理

**主要结果**：一笔明确交易的合同、发票、税务、入账和凭证处理说明。

- [ ] 确认交易主体、标的、金额、币种和资金路径。
- [ ] 说明适用发票或其他凭证。
- [ ] 说明主要税务处理和申报时点。
- [ ] 说明账务记录与材料留存。
- [ ] 列出交易完成后的核对清单。

复杂跨境交易、股权交易、重组和正式鉴证事项转游骑兵。

### `OPC/FIN/005` 经营利润与现金流诊断

**主要结果**：一份面向创业者的利润、现金和行动结论。

- [ ] 限定最近三至六个月的经营数据。
- [ ] 区分可持续收入、一次性收入和未回款。
- [ ] 识别固定成本、变动成本和主要低效支出。
- [ ] 估算产品或项目利润及安全现金周期。
- [ ] 给出优先处理的现金和经营动作。

融资顾问、估值、投资建议和复杂管理会计项目不属于本服务。

## 19. 人力资源

人力资源方向聚焦超级个体使用人员时的五个关键节点：选择关系、首次用工、外部协作、在岗检查和结束关系。

### `OPC/HR/001` 用工方式判断

**主要结果**：针对一个明确岗位或合作需求形成关系选择和风险说明。

- [ ] 比较正式员工、非全日制、劳务、外包和供应商等关系。
- [ ] 检查实际管理、报酬和工作方式。
- [ ] 说明推荐关系的成立条件。
- [ ] 列出必要文件和管理边界。
- [ ] 标记可能形成劳动关系或关系混同的风险。

境外雇佣、高管、合伙人和股权激励转游骑兵。

### `OPC/HR/002` 首次用工专项

**主要结果**：使一名首位员工在固定范围内完成规范入职。

- [ ] 确认岗位、薪酬、试用期和工作地点。
- [ ] 形成录用条件和劳动合同。
- [ ] 处理保密、成果归属和数据访问。
- [ ] 整理入职材料与社保提醒。
- [ ] 交付入职和权限开通清单。

特殊工时、境外用工和历史用工问题转游骑兵。

### `OPC/HR/003` 外包与长期协作者专项

**主要结果**：为一次明确外部合作建立工作、交付、权属和退出边界。

- [ ] 判断关系是否适合外包或独立合作。
- [ ] 明确工作范围、里程碑、费用和验收。
- [ ] 处理代码、内容、数据和其他成果归属。
- [ ] 处理保密、账号和系统权限。
- [ ] 明确终止、资料和权限回收。

可能已经形成劳动关系、多人长期混同或重大技术合作转游骑兵。

### `OPC/HR/004` 在岗用工风险体检

**主要结果**：一份现有少量员工或协作者的风险报告和整改优先级。

- [ ] 检查合同、实际工作和报酬的一致性。
- [ ] 检查薪酬、社保、试用期、休假和考勤。
- [ ] 检查员工与外包关系是否混同。
- [ ] 检查账号、数据、设备和成果权限。
- [ ] 给出补正文件和整改顺序。

工伤、仲裁、群体性问题和重大历史欠缴情形转游骑兵。

### `OPC/HR/005` 离职与解除预案

**主要结果**：针对一名员工或协作者形成退出路径、交接清单和固定范围文件。

- [ ] 梳理关系、事实和已有文件。
- [ ] 比较自愿离职、协商解除或合同终止等路径。
- [ ] 整理沟通、结算和交接顺序。
- [ ] 处理账号、设备、资料和权限回收。
- [ ] 制作适用的基础离职或解除文件。

单方辞退、裁员、工伤、仲裁和高度争议事项转游骑兵。

## 20. 知识产权

知识产权方向覆盖品牌可用性、权利取得、权属链和第三方资产使用，不以提交申请数量作为主要价值。

### `OPC/IP/001` 核心品牌保护方案

**主要结果**：一个核心品牌或产品名称的可用性、保护范围和申请优先级方案。

- [ ] 完成基础近似检索和名称使用风险检查。
- [ ] 说明建议保护的类别和项目。
- [ ] 检查核心域名与主要平台名称。
- [ ] 给出申请顺序和预算优先级。
- [ ] 列出需要保留的品牌使用证据。

多法域布局、驰名商标和复杂在先权利问题转游骑兵。

### `OPC/IP/002` 商标确权与维护

**主要结果**：完成一项明确的商标注册、续展、变更或转让事项。

- [ ] 在一个服务页面中提供有限的明确版本。
- [ ] 确认权利主体、商标、类别和办理事项。
- [ ] 检查基础申请或维护材料。
- [ ] 提交并跟踪标准流程。
- [ ] 交付官方文件和后续维护提醒。

异议、无效、驳回复审和侵权争议转游骑兵。

### `OPC/IP/003` 软件与内容确权

**主要结果**：使一项软件或内容成果形成明确权属记录和标准确权材料。

- [ ] 确认成果、创作者和权利主体。
- [ ] 检查软件、文字、图片、音视频等成果的登记适用性。
- [ ] 整理创作、版本和首次发布证据。
- [ ] 完成适用的著作权登记或确权文件。
- [ ] 说明后续使用、授权和维护方式。

专利、联合创作争议和复杂国际登记转游骑兵。

### `OPC/IP/004` 成果权属链审查

**主要结果**：一项核心产品或内容资产的权属链图、缺口和补正方案。

- [ ] 识别创始人、员工、外包和合作方贡献。
- [ ] 检查现有合同、转让和授权文件。
- [ ] 检查代码仓库、域名和平台账号归属。
- [ ] 标记权利中断、范围不清和主体不一致。
- [ ] 交付补正文件或下一步处理清单。

已发生权属争议、多人重大联合开发和跨境权利冲突转游骑兵。

### `OPC/IP/005` 第三方授权与开源合规审查

**主要结果**：一项产品使用外部代码、模型、数据和素材的合规清单。

- [ ] 盘点开源软件和主要许可证。
- [ ] 盘点字体、图片、音乐、视频和其他素材。
- [ ] 检查模型、数据集、API 和第三方代码的使用条件。
- [ ] 标记商用、署名、再分发和开放源码义务。
- [ ] 给出保留、替换、补充授权或披露的建议。

复杂许可证冲突、正式侵权意见和重大技术并购审查转游骑兵。

## 21. 数据与数字合规

数据与数字合规是面向技术型超级个体的独立专业领域，不承担云服务商或政务平台已经提供的普通备案代办。

### `OPC/DATA/001` 数字产品合规体检

**主要结果**：一个网站、App、小程序、AI 产品或 SaaS 的合规风险报告和上线整改优先级。

- [ ] 检查主体、产品功能和目标用户。
- [ ] 检查用户规则、付费、退款和公开信息。
- [ ] 检查数据收集、第三方服务和主要安全措施。
- [ ] 检查 AI 功能、内容和消费者权益风险。
- [ ] 判断必要备案、标识或许可是否适用。

本服务只做诊断；多模块整改进入数字业务或行业上线基础设施。

### `OPC/DATA/002` 个人信息处理与隐私专项

**主要结果**：一个产品的个人信息处理清单、界面提示和隐私文本。

- [ ] 盘点收集的个人信息及处理目的。
- [ ] 检查必要性、合法性基础和敏感信息。
- [ ] 盘点 Cookie、SDK 和第三方共享。
- [ ] 明确保存期限和用户权利入口。
- [ ] 形成隐私政策及必要界面提示。

大规模敏感个人信息、未成年人和正式个人信息保护影响评估转游骑兵。

### `OPC/DATA/003` AI 功能合规审查

**主要结果**：一个明确 AI 功能的数据、标识、协议和监管适用性检查结果。

- [ ] 明确输入、输出、模型和用户场景。
- [ ] 判断生成、深度合成或推荐功能的适用规则。
- [ ] 检查显式、隐式标识和用户声明。
- [ ] 检查用户协议、日志和第三方模型责任边界。
- [ ] 判断备案、安全评估或进一步专业处理是否适用。

强监管用途、正式备案、重大模型安全风险和监管沟通转游骑兵。

### `OPC/DATA/004` 第三方数据供应链审查

**主要结果**：外部 SDK、云服务、分析工具、模型 API 和其他处理方的数据流与风险清单。

- [ ] 识别发送给第三方的数据及处理目的。
- [ ] 检查存储地区、保留期限和继续转交。
- [ ] 检查数据是否用于训练、分析或画像。
- [ ] 对照合同、产品设置和隐私政策。
- [ ] 给出保留、关闭、替换或补充披露的建议。

复杂安全审计、供应链事件和重要数据处理转游骑兵。

### `OPC/DATA/005` 数据跨境适用性预审

**主要结果**：一条明确数据路径的跨境事实图、风险等级和升级处理建议。

- [ ] 识别境内外主体、系统、人员和服务商。
- [ ] 说明数据类型、规模、敏感程度和处理目的。
- [ ] 绘制数据传输、存储和远程访问路径。
- [ ] 判断是否存在跨境行为及主要适用条件。
- [ ] 说明是否需要进一步评估、合同、申报或专业意见。

正式跨境数据申报、重要数据和复杂多法域安排转游骑兵。

## 22. 专项服务与其他入口的边界

### 进入基础设施

- 一个问题需要两个以上专业模块持续编排。
- 用户需要获得可运行状态，而不只是报告、文本或一次处理。
- 需要建设流程、维护日历和多个相互依赖的成果。

### 转交游骑兵

- 事实、目标或风险无法在收款前确定。
- 需要个案策略、复杂谈判、正式法律或税务意见。
- 已经发生重大争议、调查、处罚、诉讼或安全事件。
- 涉及强监管行业、复杂跨境、多层主体或高额交易。

### 作为免费内容

- 云平台、政务平台或主管机构已经提供成熟免费办理入口。
- 用户只需要标准步骤、材料清单或入口链接。
- OPC 无法在专业判断或结果质量上提供额外价值。

## 25. 游骑兵协会

### 25.1 定位与边界

游骑兵协会是经过基本核验的外部独立顾问名录。Vault2077 只负责展示顾问本人确认的公开职业档案和联系方式，不参与后续咨询、范围确认、定价、付款、交付或争议处理。

网站只承担三个任务：

1. 帮助用户按照顾问身份找到合适的人；
2. 展示建立初步判断所需的最少职业信息；
3. 让用户直接联系顾问本人。

不建设站内预约、询价、报价、付款、订单、聊天、评价、排名、客服转接或咨询过程追踪。

### 25.2 顾问身份

游骑兵不按专项服务的专业领域分类，而按顾问本人的主要职业身份分类。首版固定为：

1. 法律顾问；
2. 财税顾问；
3. 知识产权顾问；
4. 创业顾问；
5. 产品顾问；
6. 品牌顾问；
7. 自媒体专家；
8. 设计师；
9. 软件工程顾问；
10. AI 开发专家。

每位顾问只能选择一个主要顾问身份，并可以拥有不超过五个专长标签。跨境、AI 产品、内容直播、商业交易等场景只作为专长标签，不产生新的身份分类。

人力资源顾问保留为后续候选；接入具有稳定职业身份和明确咨询能力的顾问后再新增，不为目录完整性设置空分类。

### 25.3 公开职业档案

首版档案保持最小化，只展示：

- [ ] 公开名称：真实姓名或本人长期使用的公开职业名称；
- [ ] 一个顾问身份；
- [ ] 不超过五十字的一句话介绍；
- [ ] 不超过五个专长标签；
- [ ] 一项可选的职业资质或代表性公开经验；
- [ ] 至少一种本人授权公开的职业联系方式；
- [ ] 联系方式最后确认时间和当前状态。

明确不收集或展示：

- 当前机构；
- 所在地区；
- 服务地区；
- 详细工作履历；
- 私人教育经历；
- 客户名单；
- 咨询价格；
- 咨询形式；
- 与建立职业联系无关的个人信息。

顾问可以提供微信二维码或公开账号、职业邮箱、个人网站、外部预约链接中的一种或多种。用户展开或点击后直接与顾问联系，不经过 OPC 统一工作人员。

### 25.4 页面结构

首版只建设一个页面：

```text
/opc/rangers
```

页面按顾问身份自然分段，不建设搜索、排名和推荐。每位顾问是一条可展开记录：

```text
公开名称
顾问身份
一句话介绍
专长标签
查看联系方式
```

展开后显示可选职业资质、公开联系方式和固定关系说明。每位顾问使用稳定锚点：

```text
/opc/rangers#ranger-slug
```

只有在顾问数量明显增加或出现独立分享需求后，才考虑建立 `/opc/rangers/[slug]`。

### 25.5 固定关系说明

页面首部和联系方式展开区域使用同一说明：

> 游骑兵为外部独立专家。Vault2077 仅展示经本人确认的公开职业档案和联系方式，不参与后续咨询、定价、付款、交付或争议处理。

外部链接应明确显示用户将离开 Vault2077。不得使用“购买咨询”“开始服务”或其他会让用户误以为 Vault2077 是咨询交付方的操作名称。

### 25.6 最小信息治理

不介入咨询服务不等于不治理公开名录。发布前必须：

- [ ] 获得顾问对公开职业档案和联系方式的明确授权；
- [ ] 核验公开名称对应的真实个人或稳定职业身份；
- [ ] 核验可选职业资质或代表性经验；
- [ ] 确认联系方式由顾问本人控制；
- [ ] 记录信息和联系方式最后确认时间；
- [ ] 提供信息纠错和移除入口。

联系方式失效、顾问要求暂停或公开信息无法继续确认时，档案停止展示联系方式。退出协会后稳定锚点可以显示“已不再列入游骑兵协会”，不得静默转给其他顾问。

### 25.7 领域记录

```text
RangerIdentityCategory
  id, slug, name, sort_order, status

RangerProfile
  id, slug, public_name,
  identity_category_id,
  one_line_intro, expertise_tags,
  optional_credential,
  consent_confirmed_at,
  contact_confirmed_at,
  status, updated_at

RangerContact
  ranger_id, type, display_text,
  qr_object_key, external_url, status
```

游骑兵联系方式不得复用 OPC 标准服务的统一联系渠道。

### 25.8 发布验收

- [ ] 每位公开顾问只有一个顾问身份和不超过五个专长标签。
- [ ] 不展示当前机构、所在地区、服务地区或无关个人信息。
- [ ] 档案已经获得本人授权并完成基本核验。
- [ ] 至少一种职业联系方式可用且由顾问本人控制。
- [ ] 用户无需经过 Vault2077 即可直接联系顾问。
- [ ] 页面不提供价格、预约、支付、评价、排名或咨询过程功能。
- [ ] 关系说明在页面首部和联系方式展开区域可见。
- [ ] 联系方式失效时不继续公开无效二维码、账号或链接。
