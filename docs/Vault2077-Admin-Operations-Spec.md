# Vault2077 后台与运营首版设计基线

> 状态：规范性文件。后台服务于极简运营，不改变公开产品的自动化原则。

## 1. 后台任务

后台只处理真实业务输入、用户报告的错误、外部依赖故障和必须人工完成的资格或专业确认。正常资讯采集、翻译、摘要、事件匹配、SiC 排名和边境榜单计算均自动完成，不进入日常人工审批队列。

首版使用一个共享密码，不建设人员账号、角色、组织、审批流、富文本搭建器或复杂数据大屏。

## 2. 权限边界

共享会话可以执行全部后台操作，因此所有写操作必须二次确认、记录操作日志并可识别目标记录。后台不能：

- 手工修改事件证据、Star 快照、Star 基线或榜单分数；
- 绕过 OPC 专业复核发布服务；
- 将私密报名字段复制到公开内容；
- 把未通过结构化校验的模型输出直接发布；
- 用后台新增任意抓取 URL 绕过来源治理。

## 3. 登录与会话

- 生产密码至少 20 个字符，由密码管理器生成，不进入代码、日志或客户端包。
- 服务端只保存 Argon2id 哈希；环境配置变更后所有旧会话失效。
- 登录按 IP 限速，连续失败触发短期锁定；响应不暴露密码是否配置。
- 会话 Cookie 必须为 `HttpOnly`、`Secure`、`SameSite=Strict`，最长 8 小时，无活动 60 分钟后失效。
- 登录、退出、密码轮换和锁定事件进入安全日志，但不保存明文密码或完整 IP。

## 4. 后台导航

固定为：运行概览、内容异常、来源组合、OPC 菜单、边境计划、系统记录。

运行概览只显示需要行动的信息：最近四个批次、失败任务、暂停来源、待处理纠错、待专业复核服务、待结算事项。正常指标不做装饰性图表。

## 5. 内容异常与纠错

公开用户只能提交误合并、信息错误、来源失效三类纠错报告。后台操作：

- 查看报告及关联事件或资讯；
- 隐藏或恢复资讯；
- 把资讯从错误事件移出并加入另一个现有事件；
- 将事件拆为两个事件；
- 更新来源失效状态；
- 触发翻译、资讯摘要或事件摘要重算；
- 关闭报告并记录处理说明。

LLM 不自动处理历史误合并。人工操作必须保留原记录、前后关联和时间；公开页面只显示当前有效结果，不暴露内部备注。

## 6. 来源组合治理

人工批准对象是“来源组合修订”，即一个参考项目或自有来源包的固定版本，而不是逐条批准其中已经明确声明的上游来源。一个来源组合修订获批后，其中具有明确原始来源标识的条目自动进入来源注册表；新增、移除或升级来源组合修订需要人工确认。

后台可查看来源组合版本、发现的逻辑发布者和通道、最近成功、失败率及暂停状态。系统可以因连续失败自动暂停单个通道；恢复可以自动健康检查或人工触发。后台不得接受请求参数中的任意 URL 直接采集。

首批来源组合固定为 2026-07-22 信源审计报告所列七组项目的固定提交，经去重和在线核验后生成运行 bundle；后续增删或升级仍遵循本节治理方式。

## 7. OPC 菜单

后台支持服务草稿、专业复核状态、修订、发布、停售和统一联系渠道。正式发布前执行字段完整性检查；价格、范围、材料、周期、成果、复核类型和生效日期任一缺失即阻止发布。

后台只记录专业负责人已确认的结果，不取代专业负责人作出法律、税务或其他专业判断。

## 8. 边境计划

后台支持查看报名和当前赛季状态；奖品捐献只提供查看待确认记录、确认并加入随机奖池、拒绝和结算前撤回。

所有排名值、最终资格、季度冠军和随机分配结果只读。每小时 Star 更新、季度结算、最终资格检查和随机奖品分配均由系统执行；后台不得修改 Star、基线、名次、指定奖品获得者或重复抽取。确认、拒绝或撤回奖品属于边境计划后台的写操作，必须明确目标奖品和结果。

## 9. 操作日志

每个写操作记录：时间、会话匿名标识、动作、目标类型、目标 ID、结果、原因和不含敏感数据的差异摘要。日志不可通过普通后台界面修改或删除。

共享密码无法证明具体操作者身份，界面必须明确这一限制；如果未来需要个人问责，再升级为独立账号，不在首版伪造操作者字段。

## 10. 失败与安全状态

- 数据库或存储不可用：后台只读并阻止写操作。
- 外部依赖失败：显示稳定错误码、最近成功时间和安全重试，不展示密钥或原始响应正文。
- 部分批次失败：保留上一公开版本，允许针对失败任务重试。
- 高风险操作冲突：以后到达的请求不得静默覆盖先前状态，返回冲突并要求刷新。
- 会话过期：写操作返回未授权，前端保留未提交表单内容并要求重新登录。

## 11. 验收标准

- 未登录请求无法获得任何后台数据。
- 暴力登录受到限速和锁定，会话符合 Cookie 安全要求。
- 正常内容发布无需人工批准；历史纠错只能由人工执行。
- 来源组合获批后，其明确来源自动进入注册表，无逐条审批负担。
- 任何公开数据修改、资格处置和服务发布都有操作日志。
- 后台无法直接写入排名数值或发布不完整的 OPC 服务。
- 共享密码、邮箱、挑战哈希、LLM 密钥和原始敏感日志不出现在客户端响应。
