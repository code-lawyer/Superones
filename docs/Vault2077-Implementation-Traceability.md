# Vault2077 实施追踪矩阵

> 状态：规范性工作台。更新时间：2026-07-23。它记录规范与当前代码的差距，不修改产品规则。

## 1. 状态定义

- **已实现并验证**：代码满足规范，具有与风险匹配的自动或人工验收证据。
- **已实现待补验证**：主要行为存在，但缺少稳定测试或完整状态验收。
- **部分实现**：只能支持演示或主流程，不能满足生产不变量。
- **未实现**：尚无对应模块或页面。
- **上线输入**：需要真实业务、法律或部署材料，不属于代码缺陷。

优先级：P0 阻止正式上线；P1 阻止对应频道从示例切换为正式；P2 可以在正式上线后迭代。

## 8. 当前实施顺序

1. **P0-数据基础**：PostgreSQL、批次 v2、独立资讯、发布版本、队列与隔离。
2. **P0-信息编排**：资讯处理、近月事件匹配、新事件门槛、真实双语详情。
3. **P0-边境网站 MVP**：四无与弹窗、自然季度、每小时排名、奖品捐献、自动结算、随机奖池和往届记录。
4. **P0-商业上线**：OPC Catalog、专业复核和真实业务输入。
5. **P0-安全运维**：后台认证、来源组合、操作日志、监控和恢复。
6. **P1-频道完善**：SiC 仓库改名、README SHA 缓存、逐来源新鲜度和生产监控；信息流详情与纠错；奖池和荣誉榜。
7. **P2-优化**：真实指标驱动的权重、性能和内容密度校准。

每完成一项必须同时更新本矩阵的状态和验收证据。只修改页面而未满足领域不变量，不能标记为“已实现”。
